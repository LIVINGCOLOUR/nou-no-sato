# 農の里 Phase 2 設計方針メモ

実機確認（2026-06）で出た方針を、Phase 2 以降の設計として整理したものです。
これらは「今は実装せず、方針として確定しておく」項目です。UIプロトタイプ（Phase 1）の範囲には含めません。

## 用語変更（Phase 1 で反映済み）

- 「畑ノート」「ノート」→「栽培記録」「記録」に統一。
- 下部ナビ・画面名・コピーを「栽培記録」に寄せた。
- 栽培記録は個人ユーザーが登録するもので、基本非公開。団体機能とは別。

## 1. 仲間ページの位置づけ

- 仲間ページは「個人同士が直接連絡を取り合う場」ではない。
- 「地域にどんな団体・活動者がいるかを知る」ための場とする。
- 主な役割は、その団体・活動者の**イベントへアクセスする入口**。
- 住所・詳細な活動場所・直接連絡先は原則出さない。
- ファーストコンタクトは個人間連絡ではなく**イベント参加経由**にする。
- 「この団体のイベントを見る」導線は残す。
- 「つながる」より「知る・イベントへ進む」画面として整理する。

## 2. 仲間カードに公式情報を追加（Phase 2）

- 団体・活動者の公式情報を表示できるようにする。
  - 公式ホームページURL
  - Instagram
  - その他公式SNS
- これらのURLは**団体自身が管理ページで登録**する（第三者による勝手な登録は不可）。
- 外部リンクは「公式リンク」として扱い、なりすまし・不適切リンクを防ぐ運用ルールが必要。

## 3. イベントと団体の双方向導線

- イベント一覧は全イベントを表示。
- 各イベントカード・詳細に「どの団体が開催しているか」を明示。
- イベント詳細 → 開催団体ページ へ移動できる。
- 団体ページ → 「この団体のイベントを見る」 へ進める。
- 導線は双方向（団体⇔イベント）。
- イベントが農の里における**安全なファーストコンタクトの場**になる。

## 4. アカウント種別の分離

誰でも自由にイベント作成・団体ページ管理ができると運用が破綻するため、種別を分ける。

| アカウント | できること | 発行 |
|---|---|---|
| 個人 | 栽培記録の作成 / イベント閲覧・参加予定 / 農法を学ぶ / 在来種を見る | 一般登録 |
| 団体 | 団体プロフィール管理 / 公式URL・SNS登録 / イベント作成・編集 | 運営審査制 |
| 運営 | 全体管理・承認 | 運営のみ |

- 栽培記録は個人ごとの機能なので、一般ユーザー全員が使える必要がある。
- イベント作成は**承認済み団体のみ**。誰でも団体アカウントを発行できるようにはしない。
- 団体登録には審査・承認フローを設ける。

## 5. 学ぶページの参考コンテンツ方針

- 農法詳細に YouTube / 外部サイトの参考リンクを載せることは可能。
- ただし誰でも自由に追加できる形にはしない。**運営確認済みのリンクのみ**掲載。
- 単なるリンク集ではなく、整理された補足資料として扱う。
- 農法情報の中身は当面このまま。将来、信頼できる内容に再構成する。
- 外部リンクは補助。アプリ本体の価値は「地域・イベント・記録・学び」に置く。
