1. 概要

1.1 目的

「農の里」は、自然農・自然栽培・有機農法・菌ちゃん農法などに関心のある人が、地元でゆるく仲間を見つけ、リアルイベントに参加し、農法を学び、自分の畑の記録を残し、在来種・固定種への関心を高めるための地域コミュニティアプリである。売り買いの場ではなく、地域の横のつながり（コミュニティ活性化）を目的とする。

1.2 対象ユーザー

区分説明
個人（主対象）自然農・有機で家庭菜園をしている人。初心者〜数年目。プランター栽培も含む。
団体・活動者地域で観察会・勉強会等を開く団体、サークル、活動者。運営の審査・承認を経て登録する。
運営イベントの確認・公開、団体の承認、在来種情報の確認・掲載を行う。

1.3 Phase 1 の位置づけ

Phase 1 は、バックエンド実装前に画面構成・主要導線・UIの雰囲気・プロダクト方針を確認するための静的UIプロトタイプである。DB・API・ログイン・認証・投稿保存などの本番機能は持たない。ただし「気になる」等の軽い操作状態のみブラウザの localStorage に保存する（3.4節）。

1.4 提供価値（中心価値）

  • 地元の仲間（個人・団体）との出会い
  • 運営登録イベントへの参加
  • 農法の学び（優劣づけしない比較）
  • 基本非公開の栽培記録
  • 在来種・固定種への関心（出典つき在来種マップ）

3. 機能仕様

3.1 画面一覧とルーティング

URLハッシュ（#/…）で画面を切り替える。未定義ルートは Not Found 画面（主要導線カード付き）を表示する。

ルート画面主な内容
#/homeホーム対象ユーザー明示のヒーロー（テーマ：自然な農業に関心がある仲間と、地元でつながる。）、CTA2つ（イベント／在来種マップ）、方針の信頼帯、はじめての方へ（3ステップ）、ほかにできること（記録・マップ）。
#/members仲間を探す匿名サマリー帯（個人数・団体数）、農法フィルタ、近くの個人（peers）、地域の団体・サークル（friends）。
#/groups/:id団体詳細活動リズム、歓迎メッセージ、「活動を受け取る」、季節の便り（読み取り専用）、団体のイベント、関わる在来種。
#/eventsイベント一覧種別フィルタ、月別グルーピング（「7月（3件）」等の見出し）、イベントカード（開催ステータス・人気マーク・気になるトグル付き）。開催済みは「開催済みのイベント」セクションに分離し、薄い表示＋「当日の様子・声を見る」リンクにする。
#/events/:idイベント詳細説明文を最上部に表示。開催ステータスバッジ、開催情報（日時・地域＋詳細案内の補足・定員と参加予定・料金・申込締切・主催・持ち物・雨天時）、当日の流れ（タイムライン）、気になるトグルと「◯人が気になる」、参加予定トグル、開催団体を見る（右下）、参加した人の声、関連在来種。開催済みの場合は申込系ボタンの代わりに終了案内（主催団体ページへの誘導）を表示。
#/learn農法を学ぶ先頭に在来種マップへの導線、農法カード8種、全農法の比較表（ひとこと/考え方/入口/注意点/比較軸。横スクロール・先頭列固定）、技法カード4種。
#/learn/:id農法詳細考え方・入口・注意点、成り立ち・提唱者、くわしい解説、比較軸（耕す/肥料/草/農薬）、出典・参考リンク（Web公開情報を当サイトが要約した旨を明記）。優劣づけしない前提の文言。
#/techniques/:id技法詳細農法を横断する技法（自家採種・コンパニオンプランツ・緑肥・ぼかし肥料づくり）。ひとこと・入口・解説・つながる農法・出典。自家採種は在来種マップ／情報提供へ相互リンク。
#/notes栽培記録一覧非公開方針の信頼リスト、記入例カード（記入例であることを明示）。
#/notes/new栽培記録作成作物・日付・ひとこと・写真（見た目のみ）の静的フォーム。保存なし。
#/native-map在来種マップLeaflet 地図（関東・福島、地域目安の円マーカー、出典確度で色分け）、在来種カード、情報提供CTA。
#/native-map/contribute在来種 情報提供運営確認つき（moderated submission）の静的フォームと掲載フローの説明。
#/native-varieties/:id在来種詳細地域・分類・別名・データ確度、背景、出典リンク、情報提供導線、関連する団体・イベント。
#/mypageマイページサンプル表示であることを明示。プロフィール、統計（参加予定／気になる／受け取り中）、参加予定イベント（日付順）、気になっているイベント、公開範囲の説明。
#/manage団体メニュー団体向け管理の入口（デモ）。
#/manage/group団体プロフィール編集団体情報・活動リズム・公式リンク等の静的フォーム。
#/manage/eventイベント登録イベント情報の静的フォーム（種別・日時・地域・定員・料金・申込締切・雨天時の扱い・当日の流れ・持ち物ほか）。承認済み団体のみ登録可能である旨を明示。

3.2 ナビゲーション

要素仕様
ヘッダー（PC）仲間／イベント／学ぶ／記録／マイページの5リンク。現在地をハイライト。
ヘッダー（モバイル）ブランド＋アカウントアイコン（マイページへの導線。980px以下で表示）。
下部ナビ（モバイル）ホーム／仲間／イベント／学ぶ／記録の5タブ（700px以下で表示）。在来種マップ系は「学ぶ」をハイライト。

3.3 ユーザー操作（軽いアクション）

コミュニティ参加の最初の一歩を小さくするため、申し込みより手前の低コスト操作を提供する。

操作対象挙動
気になるイベントトグル。押すとマイページの「気になっているイベント」に集まり、詳細ページの「◯人が気になる」が即時に±1される。
参加予定に入れるイベントトグル（詳細ページの主ボタン）。押すと「参加予定に入れました」に変わり、参加予定人数の表示が即時に+1される。マイページの「参加予定イベント」に日付順で集まる。実際の申込確定は主催団体からの案内で行う想定である旨を注記。
活動を受け取る団体トグル。マイページに受け取り中団体として表示（通知処理はデモのためなし）。
イベントに誘う個人（peers）トグル。より深いやりとり（連絡先交換）はイベントで直接会ったときに個人間で行う方針を注記で明示。
絞り込み仲間（農法）／イベント（種別）チップ型の button（aria-pressed 付与）。モバイルでは横スクロール。選択はセッション内のみ保持。

3.4 状態の保存（localStorage）

項目内容
キーnounosato:ui
保存対象interested（気になるイベントID）、joined（参加予定イベントID）、following（受け取り中団体ID）、invited（誘った個人ID）の4セットのみ
保存タイミングトグル操作時に保存、起動時（DOMContentLoaded）に読み込み
保存しないものフィルタ選択、栽培記録、プロフィール、各フォームの入力内容
異常時読み書きに失敗しても（プライベートモード等）初期状態で継続する

3.5 イベントの表示ロジック

項目仕様
開催ステータス今日の日付（基準年 EVENT_YEAR = 2026）と開催日・締切・残席から自動判定：終了（開催日が過去）／締切間近（申込締切まで5日以内）／残りわずか（残席2以下）／募集中。一覧カード・詳細にバッジ表示。開催済みは人気マーク非表示。
人気マーク参加予定数が定員の70%以上、または「気になる」数が12件以上のとき「人気」バッジを一覧カードと詳細に表示する（参加喚起）。
気になる人数表示値 ＝ データ上の interestedCount ＋（自分が押していれば1）。トグルで即時更新。
参加予定人数定員欄に「定員◯名（参加予定 ◯名）」の形式で表示。表示値 ＝ データ上の attending ＋（自分が参加予定に入れていれば1）。トグルで即時更新。
月別グルーピング一覧では開催予定を日付昇順に並べ、月ごとの見出し（件数付き）でまとめる。開催済みは日付降順で別セクション。
当日の流れ・雨天時全イベントが schedule（時刻＋内容のタイムライン）と rainPolicy（雨天時の扱い）を持ち、詳細に表示する。
地域の表示市町村程度＋「（詳細な場所は参加確定後にご案内）」を併記。正確な住所は掲載しない。
料金・締切料金（無料／金額）、申込締切日を開催情報に表示。
種の交換会イベント種別のひとつ。seedExchange フラグを持つイベントの詳細に「約束事」（在来種・固定種中心／登録品種の種苗は扱わない／販売禁止・無償交換／来歴を添える）を常時表示。在来種マップ・自家採種ページと相互リンク。

3.6 在来種マップ

項目仕様
地図Leaflet。対象範囲は関東地方全域＋福島県。スクロールズーム無効。全マーカーに fitBounds（maxZoom 10）。
マーカー地域目安の円マーカー。緑＝公的DB・地域資料、オレンジ＝調査中・本人確認前。ポップアップから詳細へ遷移。
位置の扱い市町村程度の概略位置のみ。正確な採種地点・圃場所在地・個人宅は扱わない。
出典各在来種に出典名・URL・データ確度ラベルを付す。
情報提供住民からの情報提供フォーム（デモ）。自由投稿ではなく、運営確認を経て確度ラベル付きで掲載する（5.2節）。

4.2 相互リンク

在来種（seeds）↔ 団体（friends）↔ イベント（events）を双方向に参照する。イベント詳細に関連在来種、団体ページに関わる在来種と主催イベント、在来種詳細に関連団体・関連イベントを表示し、「知る」と「集まる」を接続する。

5. 運用仕様

5.1 イベント・団体の登録

  • イベントを作成できるのは運営の審査・承認を経た団体のみ。一般ユーザーが自由に作成する導線は設けない。
  • イベントは運営確認のうえ公開される。
  • 公式リンクは団体自身が管理ページで登録する。第三者による登録は不可。
  • 詳細住所・正確な開催地点は、参加確定後に運営から案内する。

5.2 在来種「情報提供」の掲載フロー

  1. 住民・関係者がフォームから情報を寄せる。
  2. 運営が出典・現存状況・地域での扱いを確認する。
  3. 確認の度合いに応じ、確度ラベル付きで掲載する（出典確認済み＝地域資料系の緑バッジ／未確認＝「調査中・本人確認前」のオレンジバッジ）。
掲載は市町村程度の地域目安のみ。栽培者個人の氏名・連絡先は扱わない。種の売買・出品の場ではない。運営確認の体制・所要時間、確度を上げる出典基準、訂正・削除フローは未確定（今後決定）。

5.3 デプロイ

  • GitHub Pages（main ブランチ直配信）。main への push で自動的に再ビルド・公開される。
  • 公開URL：https://livingcolour.github.io/nou-no-sato/

6. 非機能・設計方針

6.1 プライバシー方針

  • 個人はニックネームと市町村程度の地域のみ表示。本名・詳細住所・畑の正確な位置は扱わない。
  • 栽培記録は基本非公開。公開タイムラインは設けない。
  • 連絡先交換はアプリ内では行わず、イベントで直接会ったときに個人間で行う。

6.2 コンテンツ方針

  • 農法比較は優劣づけではなく、条件・考え方・続けやすさ・地域差の理解のためのもの。
  • 守りの注記（〜しません）と歓迎の温度（見学OK・途中参加OK等）を並走させる。
  • 参考画像・外部サイト画像は直接コピーしない。

6.3 UI/UX方針

  • 緑・白・淡い生成り・少量のオレンジ。山・畑・空を想起する明るい配色。
  • スマホファースト。白いカード・やわらかい影・丸みのあるボタンで安心感。
  • 初心者が安心して眺められる密度。業務UIにしない。
  • ユーザー向け文言に開発用語（Phase 1、プロトタイプ等）を出さない。

6.4 対応環境

  • モダンブラウザ（Chrome / Edge / Safari / Firefox）。スマートフォン〜PC（ブレークポイント 980px / 700px / 360px）。
  • JavaScript 必須。localStorage が使えない環境でも閲覧・操作は可能（状態が保存されないのみ）。

7. スコープ外（作らないもの）

7.1 Phase 1 では実装しない

ログイン、ユーザー登録、DB、API、投稿保存、AI相談、DM、リアルタイムチャット、ランキング、課金、農産物販売、SNS投稿、複雑な権限管理。

7.2 除外（Not planned・今後も実装候補にしない）

困りごとから探す、条件付き相談、Q&A、通報・ブロック、季節チャレンジ、仮想畑。

除外理由：Phase 1 の中心価値から外れる。Q&A・相談導線は農法間の対立、回答品質の担保、運用負荷、差別化弱化につながる。季節チャレンジ・仮想畑はアプリの焦点をぼかす。

8. 今後の拡張候補（Phase 2 以降）

  • 運営管理のイベント登録・公開フロー（承認基準の明文化を含む）
  • ユーザープロフィール登録・本人の「気になる」等のアカウント同期
  • 栽培記録の非公開保存（サーバーサイド）
  • 農法コンテンツのCMS化
  • 在来種データの運営確認済みデータベース化と訂正・削除フロー
  • 市町村単位のゆるい検索
  • 通知とお気に入り機能
  • 管理者向けの承認・非表示機能
