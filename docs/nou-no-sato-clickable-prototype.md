# 農の里 クリック可能プロトタイプ

## 目的

1ページ紹介画面だったPhase 1静的UIを、hash routingで画面遷移を確認できるクリック可能プロトタイプにしました。

DB、API、ログイン、認証、投稿保存はありません。すべて静的データで画面遷移と導線を確認するためのものです。

## 作成した画面

- S01 ホーム
- S02 仲間を探す一覧
- S04 イベント一覧
- S05 イベント詳細
- S06 農法を学ぶ一覧
- S07 農法詳細
- S08 畑ノート一覧
- S09 畑ノート作成・編集
- S11 在来種マップ
- S12 在来種詳細
- S13 マイページ

## 初期は独立画面にしないもの

- S03 仲間プロフィール詳細
  - 仲間一覧内のカード展開で代替します。
- S10 畑ノート詳細
  - 畑ノート一覧カードと作成・編集画面で代替します。
- S14 設定・公開範囲説明
  - マイページ内の公開範囲説明カードで代替します。

## 除外 / Not planned

以下は今後の実装候補にしません。

- 困りごとから探す
- 条件付き相談
- Q&A
- 通報・ブロック
- 季節チャレンジ
- 仮想畑

除外理由は、地域の仲間、運営登録イベント、学び、非公開記録、在来種マップという中心価値に集中するためです。

## 確認方法

ローカルサーバーで確認する場合:

```powershell
python -m http.server 8000
```

```text
http://localhost:8000/index.html#/home
```

ファイルを直接開く場合もhash routingで遷移確認できます。

## 確認観点

- ホームから主要画面へ移動できるか。
- イベント詳細、農法詳細、在来種詳細へ遷移できるか。
- 畑ノートが基本非公開で、位置情報や詳細な畑住所を扱わないことが伝わるか。
- 農法比較が優劣ではなく違いとして読めるか。
- 仲間カードが詳細画面ではなくカード展開で十分か。
- マイページ内の公開範囲説明でS14相当を代替できているか。
