# 農の里 決定事項

## 決定済みのプロダクト方針

- 「農の里」は、自然な栽培に関心のある人が地元でゆるくつながる地域コミュニティアプリです。
- 中心価値は、地元の仲間、リアルイベント、学び、畑ノート、在来種・固定種への関心です。
- AI相談アプリではありません。
- Q&A掲示板、相談導線、SNS的な投稿拡散は作りません。
- 人間関係トラブルを増やしやすい機能は作りません。
- 困りごとから探す、条件付き相談、Q&A、通報・ブロック、季節チャレンジ、仮想畑は除外 / Not plannedです。

## Phase 1で作るもの

- ホーム
- 地元の仲間を探す導線
- 運営登録イベントを見る導線
- 農法を学ぶ比較表示
- 畑ノートを記録する画面イメージ
- 在来種マップを見る画面イメージ
- マイページ簡易イメージ
- スマホファーストの下部ナビ
- 画面確認用のダミーデータ

## 作らないもの

- 本格的なログイン
- ユーザー登録
- 投稿保存
- DB設計
- API実装
- AI相談
- 困りごとから探す
- 条件付き相談
- Q&A
- 通報・ブロック
- 季節チャレンジ
- 仮想畑
- リアルタイムチャット
- DM
- アバター
- 仮想広場
- ランキング
- ポイント課金
- アイテム課金
- 農産物販売
- SNS投稿
- 複雑な権限管理

## 除外 / Not planned の理由

- Phase 1の中心価値から外れるため。
- 地域の仲間、運営登録イベント、学び、非公開記録、在来種マップに集中するため。
- Q&Aや相談導線は、農法間の対立、回答品質の担保、運用負荷、差別化弱化につながるため。
- 季節チャレンジや仮想畑は、アプリの焦点をぼかすため。

## 「やさいの背景」との違い

「やさいの背景」は、売場の野菜にQRを付け、野菜が育った背景動画を見せる体験型アプリです。

「農の里」は、自然農・自然栽培・有機農法などに関心のある人が、地元で仲間を見つけ、学び、記録し、リアルイベントへ参加できる地域コミュニティアプリです。

この2つは別アプリとして扱います。

## `kotoba` とは別リポジトリで管理する方針

- 「農の里」は `kotoba` とは完全に別プロジェクトです。
- 以前 `kotoba` リポジトリにPRを作成しましたが、mergeせずcloseしました。
- 今後の正しいリポジトリは `https://github.com/LIVINGCOLOUR/nou-no-sato` です。
- `kotoba` のブランチやPRを正本として扱わないでください。

## UI/UX方針

- 緑、白、淡い生成り、少量のオレンジを使います。
- 山、畑、空を想起する明るい配色にします。
- スマホファーストで設計します。
- 白いカード、やわらかい影、丸みのあるボタンで安心感を出します。
- 自然写真が入る前提の鮮やかな雰囲気にします。
- 初心者が安心して眺められる密度にします。
- 農業専門家向けの硬い業務UIにはしません。

## 個人情報・位置情報・畑ノート公開範囲の方針

- 仲間探しでは本名を前面に出しません。
- 詳細住所や畑の正確な場所は表示しません。
- 地域表示は市町村程度に留めます。
- 畑ノートは基本非公開です。
- 位置情報や詳細な畑住所を扱わない想定です。
- DMやリアルタイムチャットは入れません。

## 在来種「情報提供」の運用フロー方針

在来種マップは、運用者だけで集めるのではなく、住民・関係者からの情報提供で少しずつ育てます。ただし自由投稿ではなく、運営確認を必須とするmoderated submissionとして扱います。

掲載までの流れ:

1. 住民・関係者がフォーム（`#/native-map/contribute`）から情報を寄せる。
2. 運営が出典・現存状況・地域での扱いを確認する。
3. 確認の度合いに応じて、データ確度ラベルを付けて掲載する。
   - 出典・公的資料が確認できたもの：`local_material`（「地域資料・要確認」など緑バッジ）。
   - 名称や現存が未確認のもの：`research_needed`（「調査中・本人確認前」オレンジバッジ）。

掲載基準・禁止事項:

- 掲載するのは市町村程度の地域目安の位置だけ。正確な採種地点・圃場所在地・個人宅・座標の詳細は登録も表示もしない。
- 栽培者個人の氏名・連絡先は扱わない。
- 出典が確認できない情報は、断定せず「調査中」として扱う。確認できるまで緑バッジにしない。
- 種の売買・出品の場ではない。販売導線は作らない。
- 第三者が他者の種情報を勝手に確定情報として登録することはできない（運営確認を経る）。

未確定の運用論点（今後決める）:

- 運営確認の担当・体制と、確認にかかる標準的な所要時間。
- 「本人確認前」から「確認済み」に上げる具体的な判断基準（必要な出典の数・種類）。
- 訂正・削除依頼の受付方法と対応フロー。

## 農法比較で優劣をつけない方針

- 自然農、自然栽培、有機農法、菌ちゃん農法、自然農法（福岡式）、パーマカルチャー、協生農法、炭素循環農法を優劣で並べません。
- 「これが唯一の正解」と見せません。
- 比較は、条件、考え方、続けやすさ、地域差を理解するためのものです。
- 農法ごとの実践条件や価値観の違いを尊重します。

## Phase 2 の技術スタック（2026-07-07 決定）

- 選定基準は「運用が安い（目標月額0円）」「シンプル（自前サーバーなし）」「一般的な構成」の3つ（優先順）。
- 構成: GitHub Pages（静的フロント・現状維持）＋ Supabase 無料枠（Postgres・Auth・RLS・Storage）。自前APIサーバー・ビルド工程は導入しない。
- 認証はメールのマジックリンク（パスワードレス）。個人／団体（運営審査制）／運営の3種別。
- 公開範囲は RLS で DB 層で強制する（栽培記録は本人のみ、詳細住所・本名・連絡先のカラム自体を作らない）。
- 設計の正は `docs/nou-no-sato-phase2-design.md` と `supabase/schema.sql`。
- 無料枠は約1週間無アクセスで一時停止するため、GitHub Actions の定期ジョブで稼働維持する。

## 農法比較の文体方針（2026-07-07 決定）

- 比較軸の文言は、ぼかした安全寄りの表現ではなく、差がはっきり見える言い切り表現を使います（例：「耕さない。」「耕す。ここが自然農との大きな違い。」）。
- 言い切るのは事実の差であり、優劣の評価は引き続き書きません。
- 比較軸は「耕す／肥料／草／農薬／機械（管理機・トラクター）／資材（マルチ等）」の6つ＋「向いているとき」。

## 農法の掲載範囲と解説の出典方針（2026-07-06 決定）

- 「不耕起・草生栽培」は特定の農法ではなく複数の農法にまたがる技法のため、農法カードから削除しました。将来「技法を学ぶ」カテゴリを作る場合の候補とします。
- （2026-07-06 追記）「技法を学ぶ」カテゴリを新設しました。自家採種・コンパニオンプランツ・緑肥・ぼかし肥料づくりの4技法を、どの農法にも属さない中立コンテンツとして掲載します。出典方針は農法と同じ（公開情報を要約・出典明記・転記しない）。自家採種は在来種マップ・情報提供と相互リンクし、「学ぶ→種をつなぐ→情報を寄せる」の循環をつくります。
- 追加した農法：自然農法（福岡式）、パーマカルチャー、協生農法、炭素循環農法。
- 「自然農」（川口由一系）と「自然農法（福岡式）」は系譜が異なるため、別カードとして扱い、混同しないよう注意書きを入れます。
- 農法解説はネット上の公開情報を広く調べたうえで、当サイトが要約・再構成して掲載します。外部サイトの文章をそのまま転記（コピー）しません。
- 各農法の詳細ページに出典・参考リンクを明記し、「公開情報を参考に当サイトが要約した」旨を表示します。
- EM農法・バイオダイナミック農法など、効果への評価が分かれるものや思想的背景があるものは、中立記述の難度が高いため当面掲載しません。

## 種の交換会の方針（2026-07-06 決定）

- イベント種別に「種の交換会」を追加しました。学んだ人が地域に還元する実地の場（学ぶ→種をつなぐ→分かち合う）として位置づけます。
- 約束事（イベント詳細に常時表示）:
  1. 在来種・固定種を中心に、自分で育てて採った種を少量ずつ持ち寄る。
  2. 品種登録された品種（登録品種）の種苗は交換に出さない（種苗法への配慮。種袋の表示などで確認）。
  3. 販売・出品の場ではない。無償の交換・お裾分けとして行う。
  4. 「いつ・どこで・どう育てたか」の来歴をひとこと添えて渡す。
- 主催は承認済み団体のみ・運営確認のうえ公開（既存イベント方針と同じ）。
- 在来種マップ・自家採種ページと相互リンクし、種を軸にした循環をつくる。

## 在来種マップの対象範囲（2026-07-06 決定）

- 対象範囲を茨城県から関東地方全域（茨城・栃木・群馬・埼玉・千葉・東京・神奈川）と福島県に拡大しました。
- 作りは従来のまま（出典つき・市町村程度の地域目安のみ・正確な採種地点や個人宅は扱わない）です。
- 新規追加分も自治体・地域団体・業界団体などの公開情報を出典として明記し、確度ラベルを付けて掲載します。
